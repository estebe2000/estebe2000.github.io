1. Introduction

Ce template de chasse au trésor est une application web interactive conçue pour créer des expériences éducatives et ludiques. Il combine des visites virtuelles, des mini-jeux, des énigmes et des éléments narratifs pour engager les utilisateurs dans une aventure immersive.

Le template est hautement personnalisable et peut être adapté à différents contextes éducatifs, culturels ou promotionnels. Que vous soyez un établissement scolaire, un musée, une bibliothèque ou une entreprise, vous pouvez utiliser ce template pour créer votre propre chasse au trésor numérique.

Note : Ce template est conçu pour fonctionner sans base de données ni serveur backend. Toutes les données sont stockées localement dans des fichiers JSON et dans le stockage local du navigateur de l'utilisateur.

3. Personnalisation

3.1 Personnalisation générale

Pour personnaliser le template selon vos besoins, commencez par ces éléments fondamentaux :

Titre et logo

  • Remplacez le logo dans images/logo.png par votre propre logo
  • Modifiez le titre du site dans toutes les pages HTML (balise <title> et <h1> dans le header)

Couleurs et thème

Les couleurs principales sont définies dans css/style.css avec des variables CSS :

:root {
    --primary-color: #2c3e50;
    --secondary-color: #3498db;
    --accent-color: #e74c3c;
    --text-color: #333;
    --background-color: #f8f9fa;
    /* ... autres variables ... */
}

Modifiez ces valeurs pour adapter le thème à votre identité visuelle.

Pied de page

Mettez à jour les informations de copyright et autres mentions dans le pied de page de chaque fichier HTML :

<footer>
    <p>© 2025 Votre Organisation - Votre Ville, Pays</p>
    <p>Description de votre projet</p>
</footer>

3.2 Personnalisation des pages

Chaque page HTML peut être personnalisée pour correspondre à votre contexte :

Page d'accueil (index.html)

  • Modifiez le texte d'introduction et la description du projet
  • Personnalisez les sections de présentation
  • Adaptez les appels à l'action

Page Histoire (histoire.html)

  • Remplacez le contenu par l'histoire de votre organisation ou lieu
  • Ajoutez des dates et événements importants
  • Incluez des photos historiques pertinentes

Page Archives (archives.html)

  • Remplacez les images et descriptions par vos propres archives
  • Organisez les archives selon vos catégories (bâtiments, événements, etc.)

Page Ressources (ressources.html)

  • Adaptez les liens et ressources à votre domaine
  • Ajoutez ou supprimez des catégories selon vos besoins
Important : Assurez-vous de maintenir la cohérence entre les pages en conservant la structure de navigation et les éléments communs.

3.3 Personnalisation des images

Remplacez toutes les images par vos propres visuels :

Images principales

  • images/logo.png : Logo de votre organisation
  • Images dans les dossiers thématiques (archives, games, etc.)

QR Codes

Les QR codes dans images/qr-codes/ doivent être générés pour pointer vers vos mini-jeux. Vous pouvez utiliser un générateur de QR code en ligne pour créer des codes qui pointent vers les URLs de vos mini-jeux.

Note : Assurez-vous que toutes les images sont optimisées pour le web (taille et poids raisonnables) pour garantir des temps de chargement rapides.

5. Mode Hacker

5.1 Activation du mode Hacker

Le mode Hacker est une fonctionnalité cachée qui ajoute une couche supplémentaire d'énigmes et de défis pour les utilisateurs plus avancés. Voici les méthodes précises d'activation :

Méthode 1 : Via la console JavaScript

  1. L'utilisateur doit ouvrir la console JavaScript (F12 ou Ctrl+Shift+J sur la plupart des navigateurs)
  2. Saisir la commande activateHackerMode() ou window.hackerMode = true
  3. Le mode hacker s'active immédiatement avec des effets visuels et sonores

Méthode 2 : Séquence de clics spécifique

  1. L'utilisateur doit cliquer sur une séquence précise d'éléments dans l'ordre suivant :
  2. Logo de l'établissement (en haut à gauche)
  3. Titre principal de la page
  4. Pied de page (deux fois)
  5. Cette séquence déclenche l'activation du mode hacker

Méthode 3 : Code secret dans l'URL

  1. Ajouter le paramètre ?hack=true à la fin de l'URL de n'importe quelle page
  2. Par exemple : https://votre-site.com/index.html?hack=true

Méthode 4 : Découverte d'indices cachés

  1. Des indices sont dissimulés dans le code source des pages
  2. En inspectant le code HTML, l'utilisateur peut trouver des commentaires avec des instructions
  3. Ces indices mènent à une séquence de touches à presser (par défaut : "h", "a", "c", "k")

Le code d'activation est défini dans js/hacker-theme.js et peut être personnalisé :

// Activation par la console
function activateHackerMode() {
    // Active le mode hacker
    document.body.classList.add('hacker-mode');
    localStorage.setItem('hackerMode', 'true');
    playHackerSound('activate');
    showHackerMessage('Accès autorisé. Mode hacker activé.');
    initMatrixRain();
}

// Activation par séquence de clics
const secretSequence = ['logo', 'title', 'footer', 'footer'];
let currentSequence = [];

// Activation par séquence de touches
const keySequence = ['h', 'a', 'c', 'k'];
let keyPressed = [];

// Vérification du paramètre URL
function checkUrlParam() {
    const urlParams = new URLSearchParams(window.location.search);
    if (urlParams.get('hack') === 'true') {
        activateHackerMode();
    }
}
Important : Vous pouvez personnaliser ces méthodes d'activation selon vos préférences en modifiant le fichier js/hacker-theme.js. Assurez-vous de mettre à jour les indices cachés dans le code HTML si vous changez les séquences d'activation.

5.2 Personnalisation du mode Hacker

Pour personnaliser le mode Hacker :

Apparence

Le style visuel du mode Hacker est défini dans css/hacker-theme.css. Vous pouvez modifier :

  • Les couleurs (généralement des tons de vert sur fond noir)
  • Les effets de glitch et de distorsion
  • Les animations et transitions

Défis et énigmes

Les défis du mode Hacker sont configurés dans plusieurs fichiers :

  • js/hacker-effects.js : Effets visuels et comportements
  • data/hacker-clues.json : Indices et astuces
  • data/hacker-phrases.json : Messages et phrases

Vous pouvez ajouter vos propres défis en modifiant ces fichiers et en ajoutant de nouvelles fonctionnalités dans js/hacker-effects.js.

5.3 Effets visuels et sonores

Le mode Hacker inclut plusieurs effets visuels et sonores pour créer une ambiance immersive :

Effets visuels

  • Pluie de code : Animation de caractères qui tombent en cascade (style Matrix)
  • Effet de glitch : Distorsion temporaire des éléments de la page
  • Terminal : Interface de terminal qui apparaît pour certaines interactions

Effets sonores

  • Sons de frappe : Lors de la saisie dans le terminal
  • Sons d'alerte : Pour les notifications et avertissements
  • Sons d'ambiance : Fond sonore électronique

Ces effets sont contrôlés par les fichiers dans js/ et les sons sont stockés dans sounds/. Vous pouvez les personnaliser ou en ajouter de nouveaux selon vos préférences.

6. Fonctionnalités principales

6.1 Visites virtuelles

La fonctionnalité de visites virtuelles permet aux utilisateurs d'explorer différents lieux en 360° et de découvrir des QR codes cachés qui débloquent des mini-jeux.

Configuration

Les visites virtuelles sont configurées dans data/virtual-visits.json. Pour chaque lieu, vous devez fournir :

  • Un identifiant unique
  • Un nom et une description
  • L'URL de la visite virtuelle à intégrer
  • L'emplacement du QR code caché
  • Le mini-jeu associé à ce lieu

Intégration des visites

Les visites virtuelles peuvent être intégrées de plusieurs façons :

  • Services tiers : Utilisez des services comme Google Street View, Matterport, ou ThingLink
  • Panoramas 360° : Intégrez des images panoramiques avec des bibliothèques comme Pannellum
  • Visites personnalisées : Créez vos propres visites avec des technologies comme A-Frame (WebVR)
Note : Si vous n'avez pas accès à des visites virtuelles professionnelles, vous pouvez utiliser des images panoramiques simples ou même des galeries d'images interactives comme alternative.

6.2 Mini-jeux

Les mini-jeux sont des défis interactifs que les utilisateurs doivent compléter pour collecter des fragments de la citation mystère.

Types de mini-jeux

Le template inclut plusieurs types de mini-jeux :

  • Puzzle : Reconstituer une image
  • Quiz : Répondre à des questions
  • Memory : Jeu de mémoire avec des paires à trouver
  • Mots croisés : Compléter une grille de mots croisés
  • Énigmes : Résoudre des énigmes textuelles
  • Recherche de mots : Trouver des mots cachés dans une grille

Ajout de nouveaux jeux

Pour ajouter un nouveau mini-jeu :

  1. Créez le fichier HTML du jeu dans un dossier approprié
  2. Ajoutez les scripts JavaScript nécessaires
  3. Configurez le jeu dans data/mini-games.json
  4. Créez un QR code pointant vers ce jeu
  5. Associez le jeu à une visite virtuelle dans data/virtual-visits.json

6.3 Citation mystère

La citation mystère est un élément central de la chasse au trésor. Les utilisateurs doivent compléter les mini-jeux pour collecter tous les fragments et reconstituer la citation complète.

Configuration

La citation est configurée dans data/quote-fragments.json :

  • Définissez l'auteur et la source de la citation
  • Divisez la citation en fragments (un par mini-jeu)
  • Assurez-vous que chaque fragment a du sens individuellement

Affichage progressif

À mesure que l'utilisateur complète les mini-jeux, les fragments sont débloqués et affichés dans la section "Citation mystère" de la page des mini-jeux. Une fois tous les fragments collectés, la citation complète est révélée et l'utilisateur peut accéder à l'épreuve finale.

6.4 Épreuve finale

L'épreuve finale est débloquée une fois que l'utilisateur a collecté tous les fragments de la citation mystère. Elle consiste généralement en une tâche créative basée sur les éléments découverts pendant la chasse au trésor.

Configuration

L'épreuve finale est configurée dans data/final-instructions.json et data/email-info.json :

  • Définissez les instructions et consignes pour l'épreuve
  • Configurez les informations pour l'envoi par email
  • Personnalisez les messages de félicitations

Personnalisation

Vous pouvez adapter l'épreuve finale à vos objectifs pédagogiques ou ludiques :

  • Rédaction : Demandez aux utilisateurs d'écrire une histoire ou un poème
  • Création artistique : Proposez de créer un dessin ou une composition
  • Réflexion : Invitez à une analyse ou une réflexion sur un thème
  • Projet : Suggérez un mini-projet lié à la thématique

7. Système de profil utilisateur

Le template inclut un système de profil utilisateur qui permet de sauvegarder la progression et les préférences de l'utilisateur localement dans son navigateur.

Fonctionnalités du profil

  • Sauvegarde de la progression : Mini-jeux complétés, fragments collectés, etc.
  • Préférences d'accessibilité : Taille de texte, contraste, etc.
  • Thèmes visuels : Choix du thème d'affichage
  • Statistiques : Temps passé, score, etc.

Implémentation

Le système de profil est implémenté dans js/core/user-profile-manager.js et utilise le localStorage du navigateur pour stocker les données. Vous pouvez personnaliser les informations stockées et les fonctionnalités du profil selon vos besoins.

Attention : Les données stockées dans localStorage sont spécifiques au navigateur et à l'appareil de l'utilisateur. Si l'utilisateur change de navigateur ou d'appareil, sa progression ne sera pas disponible.

8. Compatibilité Mobile et PC

Le template est entièrement responsive et s'adapte automatiquement à tous les types d'appareils, des smartphones aux ordinateurs de bureau.

Fonctionnalités responsive

  • Navigation adaptative : Menu hamburger sur mobile, navigation complète sur desktop
  • Mise en page fluide : Utilisation de grilles CSS et de media queries pour adapter le contenu
  • Images responsives : Optimisation des images pour différentes tailles d'écran
  • Contrôles tactiles : Support des interactions tactiles pour les appareils mobiles
  • Orientation : Adaptation à l'orientation portrait et paysage sur mobile

Éléments spécifiques aux mobiles

  • Menu overlay : Navigation plein écran sur mobile pour une meilleure expérience tactile
  • Taille des boutons : Éléments interactifs dimensionnés pour être facilement utilisables au doigt
  • Simplification : Certains éléments complexes sont simplifiés sur mobile pour une meilleure lisibilité

Test de compatibilité

Le template a été testé sur les configurations suivantes :

  • Navigateurs desktop : Chrome, Firefox, Safari, Edge
  • Navigateurs mobiles : Chrome pour Android, Safari pour iOS
  • Tailles d'écran : De 320px (petit mobile) à 1920px (grand écran)
Note : Bien que le template soit optimisé pour tous les appareils, certaines fonctionnalités avancées comme les visites virtuelles 360° peuvent nécessiter plus de ressources sur les appareils mobiles. Assurez-vous de tester ces fonctionnalités sur différents appareils.

9. Déploiement

Le template peut être déployé sur n'importe quel serveur web statique. Voici les étapes générales pour déployer votre chasse au trésor personnalisée :

Préparation

  1. Personnalisez tous les fichiers selon vos besoins
  2. Optimisez les images et autres ressources
  3. Testez localement pour vous assurer que tout fonctionne correctement

Options de déploiement

  • Hébergement statique : GitHub Pages, Netlify, Vercel, etc.
  • Serveur web traditionnel : Apache, Nginx, etc.
  • Hébergement partagé : Services d'hébergement web classiques

Étapes de déploiement (exemple avec GitHub Pages)

  1. Créez un dépôt GitHub pour votre projet
  2. Poussez tous vos fichiers vers ce dépôt
  3. Activez GitHub Pages dans les paramètres du dépôt
  4. Votre site sera disponible à l'adresse https://votre-nom-utilisateur.github.io/nom-du-depot
Note : Si vous utilisez des visites virtuelles ou des services tiers, assurez-vous qu'ils sont accessibles depuis votre domaine de déploiement (problèmes potentiels de CORS).

10. Conseils et bonnes pratiques

Conception

  • Cohérence thématique : Maintenez une cohérence visuelle et narrative dans toute l'expérience
  • Difficulté progressive : Commencez par des défis simples et augmentez progressivement la difficulté
  • Indices clairs : Fournissez des indices suffisamment clairs pour guider les utilisateurs sans frustration excessive
  • Feedback immédiat : Donnez un retour immédiat aux actions des utilisateurs

Technique

  • Optimisation des ressources : Compressez les images et autres médias pour des temps de chargement rapides
  • Tests multi-navigateurs : Testez sur différents navigateurs et appareils
  • Sauvegarde régulière : Sauvegardez régulièrement la progression des utilisateurs
  • Accessibilité : Assurez-vous que votre chasse au trésor est accessible à tous les utilisateurs

Pédagogie

  • Objectifs clairs : Définissez clairement les objectifs pédagogiques de chaque activité
  • Contenu adapté : Adaptez le contenu au niveau et aux intérêts de votre public cible
  • Équilibre ludique/éducatif : Trouvez un bon équilibre entre divertissement et apprentissage
  • Réflexion finale : Incluez une phase de réflexion ou de synthèse à la fin de l'expérience
